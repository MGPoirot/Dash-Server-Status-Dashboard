# Dash — Server Status Dashboard

Dash is a lightweight, file-driven dashboard for monitoring the status and health of services, hosts, and integrations. It is designed to be simple to operate, transparent in how data is represented, and easy to extend without requiring a database or complex backend.

At its core, Dash visualizes metrics that are defined declaratively and updated externally. Each metric represents a single observable concept (for example: “Watchtower running”, “Disk usage”, or “Last backup time”) and is rendered consistently across the dashboard.

Dash deliberately separates:

* **What a metric means** (its definition),
* **How it evolves over time** (its series),
* **What its current value is** (its latest state),
* **What happens when it changes** (integrations).

This separation allows metrics to be generated by any system capable of writing JSON files, while the dashboard itself remains static, predictable, and easy to reason about.

The intended use cases include:

* Home servers and homelabs
* Small infrastructure setups without centralized monitoring
* Side projects where operational visibility is useful but heavy tooling is undesirable

Dash favors explicit configuration over convention and avoids hidden behavior. If a metric appears on the dashboard, it exists as a file on disk.
